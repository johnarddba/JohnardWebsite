Foglight for Databases – DBA User Guide

1. Introduction

Foglight is a performance monitoring and diagnostic tool designed to help Database Administrators (DBAs) proactively manage database environments. It provides visibility into performance, availability, and resource usage across multiple platforms (SQL Server, Oracle, MySQL, DB2, etc.).

This guide explains how DBAs can use Foglight in daily operations and highlights its benefits.

2. Key Features for DBAs

  • Cross-platform support – SQL Server, Oracle, MySQL, Postgres, DB2, etc.
  • Real-time monitoring – CPU, memory, sessions, and I/O.
  • Historical data analysis – Trend reporting and baselines.
  • Alerting & notifications – Custom thresholds and alerts.
  • Root cause diagnostics – Identify bottlenecks faster.
  • Capacity planning – Forecast growth and plan upgrades.

3. How to Use Foglight as a DBA

Step 1: Accessing Foglight Console

Open the Foglight Web Console, log in with DBA credentials, and navigate to the Databases Dashboard.

Step 2: Adding Database Instances

Go to Administration → Agents → Database Agents, add database instances, and verify connection.

Step 3: Monitoring Day-to-Day Operations

  • Health Overview Dashboard – Check overall DB health and alerts.
  • Instance Dashboard – Drill into CPU, memory, and sessions.
  • Alarms & Alerts – Monitor blocking sessions, deadlocks, and disk usage.

Step 4: Troubleshooting

Use Top SQL View, Wait Events, and Execution Plans to identify performance issues.

Step 5: Capacity & Growth Planning

Analyze historical trends to forecast growth and resource needs.

4. Benefits for Day-to-Day DBA Operations

Task Challenge without Foglight Benefit with Foglight
Performance Monitoring Manual checks via T-SQL/DMVs. Proactive alerts and real-time insights.
Query Tuning Manual profiling required. Top SQL report shows high-impact queries instantly.
Resource Management Hard to track across servers. Centralized dashboard.
Blocking/Deadlocks Missed unless monitoring live. Automated detection and drill-downs.
Capacity Planning Risk of unexpected outages. Predictive growth analysis.
Multi-DB Support Switching between tools. Single console view.
Incident Response Manual log checks cause delays. Immediate alerts and RCA.

5. Best Practices

  • Configure custom thresholds aligned with SLAs.
  • Schedule regular reports (uptime, performance trends).
  • Integrate with ticketing systems (ServiceNow, Jira, etc.).
  • Review baseline performance regularly.
  • Enable OS-level monitoring (CPU, RAM, disk).

6. Conclusion

Foglight helps DBAs move from reactive firefighting to proactive database management. It simplifies monitoring, accelerates troubleshooting, and ensures performance, availability, and capacity remain under control, reducing downtime and improving user satisfaction.
