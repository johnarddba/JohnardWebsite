Who this is for

DBAs moving to AWS, or experienced database engineers who want a structured roadmap to deploy, secure, and operate databases in the cloud. Each week includes key topics, a hands-on lab, and a quick checkpoint to confirm understanding.

Week 1: AWS Core Foundations Fundamentals

Key Topics

  • AWS Global Infrastructure: Regions, Availability Zones, and Edge Locations.
  • Identity & Access Management (IAM): roles, policies, least-privilege for DBAs.
  • Networking for databases: VPC, subnets, security groups, NACLs.
Hands-On Lab: Create a VPC with private/public subnets and an IAM role scoped for database access.
Checkpoint: Explain the difference between Multi-AZ and Multi-Region architectures in your own words.

Week 2: Amazon RDS Deep Dive Relational

Key Topics

  • Engines: SQL Server, MySQL, PostgreSQL, Oracle, MariaDB.
  • Backups & Restore: automated backups, snapshots, point-in-time recovery (PITR).
  • Observability: Enhanced Monitoring, Performance Insights, CloudWatch metrics.
Hands-On Lab: Deploy an RDS SQL Server Multi-AZ instance. Configure automated backups and perform a snapshot restore test.
Checkpoint: Create a CloudWatch alarm for CPU > 80% on the instance.

Week 3: Amazon Aurora Cloud-Native

Key Topics

  • Architecture: decoupled storage, 6-way replication, fast failover.
  • Provisioned vs. Serverless v2 and autoscaling behavior.
  • Global Database and cross-region replication patterns.
Hands-On Lab: Deploy an Aurora cluster with at least one reader. Simulate an instance failure to observe failover.
Checkpoint: Compare Aurora vs. RDS SQL Server for availability, performance, and cost.

Week 4: High Availability & Disaster Recovery Resilience

Key Topics

  • Multi-AZ failover vs. Multi-Region DR.
  • Aurora Global Database for low-RPO/RTO scenarios.
  • SQL Server Always On Availability Groups (EC2 vs. RDS).
Hands-On Lab: Build a SQL Server Always On AG on EC2 across two AZs. Perform a planned and unplanned failover test.
Checkpoint: Draft a DR plan for a 2‑TB SQL Server database including RPO/RTO targets and cross-region strategy.

Week 5: Monitoring, Security & Compliance Ops

Key Topics

  • Monitoring stack: CloudWatch, CloudTrail, AWS Config.
  • Encryption: KMS at rest; SSL/TLS in transit.
  • IAM database authentication and access boundaries.
Hands-On Lab: Enable RDS audit/slow logs; create a KMS-encrypted RDS instance; validate TLS connections.
Checkpoint: Describe how CloudTrail supports DBA auditing and forensics.

Week 6: Migrations & Hybrid Scenarios Move & Modernize

Key Topics

  • Database Migration Service (DMS): tasks, endpoints, replication instance sizing.
  • Schema Conversion Tool (SCT) for heterogeneous migrations.
  • Cutover planning: one-time vs. continuous replication and downtime windows.
Hands-On Lab: Use DMS to migrate a sample on‑prem SQL Server to RDS. Try SCT to convert a SQL Server schema to Aurora PostgreSQL.
Checkpoint: List three risk areas in heterogeneous migrations and how you would mitigate each.

Week 7: NoSQL & Analytics Broaden

Key Topics

  • DynamoDB: partition design, adaptive capacity, Global Tables.
  • Amazon Redshift: columnar storage, WLM/concurrency scaling, RA3 nodes.
  • Data lake tooling: AWS Glue, Amazon Athena, Lake Formation.
Hands-On Lab: Create a DynamoDB table using on‑demand capacity; run Athena queries over S3 data cataloged by Glue.
Checkpoint: Identify scenarios where DynamoDB is a better fit than RDS.

Week 8: Automation & Cost Optimization Scale

Key Topics

  • Infrastructure as Code: AWS CLI, CloudFormation, Terraform for DB stacks.
  • Cost levers: Reserved/Save Plans, Aurora Serverless, right‑sizing, storage choices.
  • Operational automation: snapshot policies, patch windows, blue/green deployments.
Hands-On Lab: Automate RDS deployment with CloudFormation or Terraform; script snapshot & restore flows via AWS CLI.
Checkpoint: Propose cost optimizations for a 24×7 production RDS SQL Server workload.

Capstone Project

Design and deploy a production-ready SQL Server on AWS that includes:

  • Multi-AZ high availability and cross-region disaster recovery.
  • End-to-end monitoring with CloudWatch and proactive alarms.
  • Automated backups with regular restore validation.
  • Cost optimization using right-sized instances and commitments.
